FullStory Pricing Page — Prospect Behavior Report
mailchimp.com/pricing · Past 60 days · Prospects only
Key Finding
Desktop prospects scroll only 43% down the pricing page — meaning the majority never reach the plan comparison table, the most conversion-critical section of the page. Mobile prospects face even more friction: FullStory has flagged layout shifts after page refresh and a JS error that triggers a full page reload for visitors arriving from social ads. Combined, these issues create a strong case for three experiments: a sticky CTA bar, a condensed mobile plan selector, and a hero-section trust-signal module to reduce early abandonment.

53 users

FullStory Opportunities — Mobile
Friction points most likely to suppress mobile conversions
Error Causes Page Reload After Social Ad Click
54 users
A JS error fires after a custom Experiment event on mobile, causing a full page reload. This hits users arriving from social media ads — a high-intent audience — at the worst possible moment.
Instrument the Experiment event error handler to fail gracefully without page reload. This fix alone could recover paid social conversion rate on mobile.
CLS After Page Refresh on Pricing
83 users
Layout shifts after reload on the pricing page cause content to jump, disrupting the plan comparison experience on mobile. One user also encountered a 406 error in the signup flow.
Pin plan card heights with CSS min-height and lazy-load non-critical assets to eliminate CLS. Target a CLS score below 0.1.
Intermittent Errors Impacting UX
111 users
Network errors and uncaught JS exceptions causing page loading delays and interaction latency across Mailchimp pages including pricing.
Conduct a mobile-specific performance audit targeting GTM and analytics tag load order — these third-party scripts appear to be blocking interactivity on low-end devices.
